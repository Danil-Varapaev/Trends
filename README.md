Данный раздел описывает порядок взаимодействия с выходными артефактами скрипта: ZIP-архивом индивидуальных отчётов и текстовым журналом аномалий.
Материал предназначен для специалистов отдела качества, микробиологической лаборатории и производственного контроля.
После успешного запуска скрипт генерирует два файла в корневой директории:
microbio_trends.zip - архив с папками по каждому помещению. Внутри находятся Excel-отчёты, разделённые по типу мониторинга и классу чистоты.
anomalies_report.txt - сводный текстовый журнал, фиксирующий все отклонения, пропуски в исходных данных и технические флаги качества.

Внутри архива структура выглядит так:
📁 output_reports/
 ┣ 📁 P-025/
  ┣ 📄 P-025_air_asp_operated_B.xlsx
 ┃ ┣ 📄 P-025_air_sed_rest_B.xlsx
 ┃ ┗ 📄 P-025_swabs_operated_B.xlsx
 ┣ 📁 CL-28/
  ┣ 📄 CL-28_air_asp_operated_D.xlsx
 ┃ ┗ ...
 ┗ 📄 trend_chart.png (в каждой папке)


Как работать с журналом аномалий (anomalies_report.txt)
Журнал служит первичным фильтром контроля качества. Он позволяет за 5–10 минут оценить состояние мониторинга без ручного просмотра сотен графиков.
Навигация и поиск:
Откройте файл в любом текстовом редакторе (Notepad++, VS Code, стандартный Блокнот).
Используйте Ctrl+F для поиска по ключевым словам:
ПРЕВЫШЕНИЕ → микробиологические отклонения
ОТБРОШЕНА / ОШИБКА ДАННЫХ → проблемы ввода в исходные журналы
🚩 ФЛАГ → автоматические эвристики качества тренда



Приоритизация действий:

ОТБРОШЕНА (пустой метод) - в исходном Excel не указан метод отбора (А/С). Скрипт не может определить тип мониторинга, строка исключена из расчётов.

ОТБРОШЕНА (нераспознанный метод) - в поле метода указаны символы, не соответствующие аспирации (А/A) или седиментации (С/C).

ОШИБКА ДАННЫХ: Отсутствует класс чистоты - для помещения в выбранном типе мониторинга ни в одной строке не указан класс A/B/C/D. Без класса невозможно применить лимиты.

КРИТИЧЕСКОЕ ПРЕВЫШЕНИЕ ПРЕДЕЛА - превышен уровень действия (action). Требует анализа тренда и корректирующих действий.

ПРЕВЫШЕНИЕ УРОВНЯ ТРЕВОГИ - превышен уровень предупреждения (warning). Сигнал для внимания, но не нарушение лимита.

🚩 ФЛАГ: Все значения = 0 - за весь период ни одной КОЕ. Может указывать на ошибку отбора/посева или аномально высокую стерильность.

🚩 ФЛАГ: Все значения >= предела - систематическое превышение. Возможна ошибка классификации помещения или хроническая контаминация.

🚩 ФЛАГ: Мало данных для квартала - менее 4 точек за период. Тренд статистически ненадёжен.




Каждая запись в журнале привязана к конкретному Excel-файлу. Для верификации:
Найдите в журнале ПОМЕЩЕНИЕ: X | ТИП: Y | КЛАСС: Z
Откройте архив → папка X → файл X_Y_Z.xlsx
Перейдите на лист Данные → найдите строку по Лаб№ и Дата


Как работать с архивом трендов (microbio_trends.zip)

Архив содержит готовые к печати и аудиту отчёты по каждому сочетанию "помещение + тип отбора + класс чистоты".

Навигация:
Распакуйте архив в рабочую директорию.
Перейдите в папку нужного помещения (например, P-025).
Выберите файл по типу мониторинга:
air_asp / air_sed → мониторинг воздуха (аспирация / седиментация)
swabs → мониторинг поверхностей (смывы)
operated / rest → состояние помещения (эксплуатация / покой)
Суффикс _A, _B, _C, _D → класс чистоты ISO/GMP


Каждый файл содержит два листа:

График - визуальный тренд, таблица Дата → КОЕ, встроенное изображение графика с линиями лимитов

Данные - полная таблица: Дата, Лаб№, Подразделение, Результат КОЕ, Уровень тревоги/действия/предела
